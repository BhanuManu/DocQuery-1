# DocQuery

DocQuery is a full-stack web application that allows users to upload documents, embed their contents, and query them using an LLM-based Retrieval-Augmented Generation (RAG) system. 

## Features
- **User Authentication**: Secure registration and login flows.
- **Document Management**: Upload and manage your documents.
- **Interactive Chat**: Query your documents and get intelligent responses generated by LLMs with context extracted directly from your files.
- **Profile Management**: View account statistics, manage your profile, and handle account deletion.
- **Dark/Light Mode**: Toggleable theme for the user interface.

## Technology Stack

**Backend (Python/FastAPI)**
- [FastAPI](https://fastapi.tiangolo.com/): High-performance web framework for APIs.
- [SQLAlchemy](https://www.sqlalchemy.org/): ORM for database interactions.
- [pgvector](https://github.com/pgvector/pgvector): PostgreSQL extension for vector similarity search.
- [Sentence Transformers](https://www.sbert.net/): Local model for generating vector embeddings from document chunks.
- [Google GenAI](https://ai.google.dev/): LLM provider for answering user queries.
- [PyPDF](https://pypi.org/project/pypdf/): Parsing and extracting text from uploaded PDF documents.

**Frontend (React/Vite)**
- [React](https://react.dev/): Component-based UI library.
- [Vite](https://vitejs.dev/): Extremely fast frontend build tool and development server.
- [React Router DOM](https://reactrouter.com/): Declarative routing for React applications.
- [Axios](https://axios-http.com/): Promise-based HTTP client for the browser.

## Prerequisites
- Node.js (v18+)
- Python (v3.10+)
- Docker & Docker Compose (for running PostgreSQL with pgvector)

## Getting Started

### 1. Start the Database
The backend relies on PostgreSQL with the `pgvector` extension for storing vector embeddings. Use Docker Compose to spin up the database:
```bash
cd backend
docker-compose up -d
```

### 2. Backend Setup
Set up your Python virtual environment and install dependencies:
```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

You'll need a `.env` file in the `backend` directory with your database credentials and API keys (e.g., `GOOGLE_API_KEY`).

Start the FastAPI server:
```bash
uvicorn main:app --reload
```
The backend will be running on `http://localhost:8000`.

### 3. Frontend Setup
Install Node dependencies and start the Vite development server:
```bash
cd docquery-ui
npm install
npm run dev
```
The frontend will be running on `http://localhost:5173`.

## Architecture
1. **Document Uploading**: Users upload PDFs, which are parsed and chunked into smaller text segments by the backend.
2. **Embedding**: Each text segment is converted into a vector embedding using `sentence-transformers`.
3. **Storage**: The original text and vector embeddings are stored in the Postgres database using `pgvector`.
4. **Querying**: When a user submits a query through the chat interface, the query is embedded, and a similarity search is performed to find the most relevant document chunks.
5. **Generation**: The retrieved chunks are provided as context to a Google Gemini LLM, which generates a natural language response based on the document data.
